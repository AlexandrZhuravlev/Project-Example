Пример тестового проекта

Для GitHub

  • git clone path-to-repositiry - путь к репозиторию
  • git status - статус текущего локального репозитория
  • git add . - добавить все файлы в текущем репозитории
  • git commit -am "added new file index.html" - закоммитить изменения во всех файлах и дать им комментарии
  • git commit -m "added new file index.html" - закоммитить изменения из индекса и дать им комментарии
  • git push - отправить закоммиченные файлы в репозиторий на сервер
  • git pull - взять файлы из репозитория

    • File change

    Working directory

  • git checkout -- "file" - не сохранять изменение файла и вернуть состояние к последнему коммиту
  • git checkout -- . - не сохранять изменение файла и вернуть состояние к последнему коммиту
  • git clean -xdf - удалить новые добавленные файлы

    • Stading Area(Index)

  • git reset -- "file" - убрать файл из индекса(STADING AREA), но отслеживать его
  • git rm -f index.html - принудительно удалить файл (стереть с диска)
  • git rm --cached index.html - удалить файл из индекса и не отслеживать его
  • git reset HEAD index.html- убрать файл из и откатить в Workong directory

    • Local branch(Commit)

  • git reset HEAD^^ (HEAD~2) - удалить коммит (~ + количество коммитов, либо ^^^)
  • git commit --amend - перезатереть последний коммит текущим

    • Remote repository

  • git revent "sha1"
  • git push --force - принудительно добавить в репозиторий коммит с текущей ветвью
  • Future changes
  • git reset --hard "f4e97f6de5cdf8f2de6acf5aaa416290e6dd53ce" - откатиться на выбранный хэш-код коммита
